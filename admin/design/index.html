Buttons

Consistent button patterns across the platform. Primary for actions, outline for secondary, ghost for inline.

Primary Buttons

bg: --color-primary, color: #fff, radius: 6-8px, weight: 600

Outline / Secondary Buttons

border: --color-primary or --color-border, bg: subtle or transparent

Modal Action Buttons

Cancel: bordered transparent. Confirm: primary filled. Always right-aligned.

Ghost / Inline Buttons

No background, semantic color, appears inline in rows

Patterns & Conventions

Behavioral patterns and naming conventions used throughout the codebase.

PatternConvention
Theme persistencelocalStorage.getItem('moonraker-theme') - reads on page load before paint
Sidebar collapselocalStorage.getItem('moonraker-sidebar') - text hidden via color: transparent
Optimistic updatesChange local state first, revert on API failure. Standard for all status changes.
Window-scoped functionsAll functions called from inline onclick must be assigned to window
Event propagationDropdowns, links, delete buttons use onclick="event.stopPropagation()" inside clickable rows
Status namingDeliverables end-state: delivered. Checklist end-state: complete. Intentionally different semantics.
Expand stateSingle expandedXxxId variable per tab. Toggled by toggleXxxExpand(id). Re-render on toggle.
Tab count badgesShow outstanding items (non-complete/non-delivered). Amber when > 0, muted when info-only.
Supabase queriesUse sb() helper for reads, apiAction() for writes. Two-step lookup for PostgREST (no subqueries).
GitHub push (large files)Write payload to /home/claude/push.json then curl -d @push.json
